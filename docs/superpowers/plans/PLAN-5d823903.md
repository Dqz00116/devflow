# PLAN-5d823903: Ralph Loop Integration Implementation Plan

## Scope

Implement Phase 1 MVP of Ralph Loop integration into DevFlow v2.0. Focus on:
- `devflow run --tool local` (subprocess agent runner)
- VCS abstraction with diff/patch checkpointing (Git + NoVCS)
- Backlog auto-generation from workflow TOML
- Progress append-only logging
- New CLI commands: `run`, `loop-status`, `sync-backlog`, `loop-reset`

Out of scope for this plan:
- External CLI support (`claude`, `amp`, `kimi`)
- SVN driver (framework in place, implementation deferred)
- `loop_behavior` TOML parsing extensions (framework in place, all steps default to `"run"`)

## Test Strategy

### Unit Tests
- `test_vcs.py`: `GitDriver` in a temp repo generates `.diff`; `NoVCSDriver` snapshots files
- `test_backlog.py`: `generate_from_workflow()` produces correct task list; `next_pending()` / `mark_done()` work
- `test_progress.py`: append logs correctly; `recent_summary(3)` returns expected lines
- `test_agent_runner.py`: invoke module via subprocess with stdin, assert expected exit code and stdout format

### Integration Tests
- `test_loop_engine.py`: set up temp project with 2-step workflow, mock `_spawn_agent` to simulate success, assert:
  - backlog tasks are marked done
  - checkpoint files are created
  - progress.md is appended
  - `WorkflowEngine` ends on correct step

### Regression Tests
- Run `pytest tests/` to ensure no breakage in `workflow_engine.py`, `gate_checker.py`, or `cli.py`

## Constraints

- `WorkflowEngine.advance()` signature must not change
- `DEVFLOW_ALLOW_SHELL=1` still required for `command_success` gates
- All subprocess calls on Windows use `creationflags=CREATE_NO_WINDOW` (when available), `encoding="utf-8"`, and prompt via stdin
- No external CLI dependencies for MVP
