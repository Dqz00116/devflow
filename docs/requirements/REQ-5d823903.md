## Description

Integrate Ralph Loop's autonomous execution model into DevFlow v2.0. The goal is to enable an agent to run `devflow run` and have the Loop Engine automatically drive the workflow step-by-step, spawning fresh AI contexts for each iteration, validating gates, and capturing progress snapshots. This reduces agent cognitive load from "remember and manually drive the entire workflow" to "complete the single task given."

## Acceptance Criteria

- [x] `devflow run` command starts an autonomous loop that runs until completion, blockage, or max iterations
- [x] Loop Engine reads `.devflow/backlog.json`, maps tasks to workflow steps, and advances via `WorkflowEngine`
- [x] VCS abstraction layer (`src/devflow/vcs.py`) supports Git, SVN, and no-VCS fallback
- [x] **Checkpoint generates diff/patch files** instead of directly committing to VCS:
  - Git → `git diff` output saved as `.devflow/checkpoints/CHECKPOINT-<id>.diff`
  - SVN → `svn diff` output saved as `.devflow/checkpoints/CHECKPOINT-<id>.patch`
  - No VCS → copy modified file list or full snapshot to `.devflow/checkpoints/`
- [x] Backlog module (`src/devflow/backlog.py`) can auto-generate tasks from a workflow TOML file
- [x] Progress logger (`src/devflow/progress.py`) appends learnings to `.devflow/progress.md` after each iteration
- [x] `devflow loop-status`, `devflow sync-backlog`, and `devflow loop-reset` CLI commands are functional
- [x] Loop respects `loop_behavior` step controls (`run`, `pause`, `skip`) and `human_in_the_loop` workflow settings
- [x] `command_success` gates during loop execution still require `DEVFLOW_ALLOW_SHELL=1`
- [x] All existing `devflow current` / `devflow done` behavior remains backward compatible
- [x] Tests cover diff checkpointing, backlog generation, and loop iteration flow
