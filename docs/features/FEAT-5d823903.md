# FEAT-5d823903: Ralph Loop Integration

## Overview

Add an autonomous Loop Engine on top of DevFlow's existing workflow system. The loop reads a backlog, spawns an agent for one step at a time, validates gates, and persists checkpoints as diff/patch files (not direct commits).

## Components

### 1. VCSDriver (`src/devflow/vcs.py`)

- `GitDriver`: generates diff via `git diff`, saves to `.devflow/checkpoints/CHECKPOINT-<sha>.diff`
- `SVNDriver`: generates diff via `svn diff`, saves to `.devflow/checkpoints/CHECKPOINT-<rev>.patch`
- `NoVCSDriver`: copies changed files snapshot to `.devflow/checkpoints/CHECKPOINT-<timestamp>/`

All drivers expose:
- `has_uncommitted_changes() -> bool`
- `get_diff_text() -> str`
- `save_checkpoint(tag: str) -> Path | None`
- `is_available() -> bool`

### 2. Backlog (`src/devflow/backlog.py`)

Schema: `.devflow/backlog.json`
```json
{
  "source": "MODE-A",
  "tasks": [
    { "id": "req-create", "workflow_id": "MODE-A", "step_id": "req-create",
      "title": "Create Requirement", "passes": false, "checkpoint_id": null }
  ]
}
```

Provides:
- `Backlog.generate_from_workflow(workflow: Workflow) -> Backlog`
- `next_pending() -> Task | None`
- `mark_done(task_id, checkpoint_id=None)`

### 3. Progress Logger (`src/devflow/progress.py`)

Appends to `.devflow/progress.md`:
```markdown
## 2026-04-16 20:45 вҖ” implement-sdd
- Completed: implement-sdd
- Checkpoint: .devflow/checkpoints/CHECKPOINT-abc1234.diff
- Notes: ...
```

Provides:
- `log(message: str)`
- `recent_summary(n: int = 5) -> str`

### 4. LoopEngine (`src/devflow/loop_engine.py`)

Core loop (`run(max_iterations=10)`):
1. Ensure backlog exists (auto-generate from current workflow if missing)
2. Pick next pending task
3. Set `current_step` in state
4. Build prompt (step prompt + recent progress summary)
5. Spawn agent (`--tool local` first)
6. Call `engine.advance()` to validate gates
7. On pass: save checkpoint diff, mark task done, log progress
8. On fail route: log routing, continue loop
9. On blocked: log blockage, pause loop

### 5. CLI Extensions (`src/devflow/cli.py`)

New commands:
- `devflow run [--tool local] [--max-iterations 10]`
- `devflow loop-status`
- `devflow sync-backlog`
- `devflow loop-reset`

## Data Flow

1. `devflow select-workflow MODE-A`
2. `devflow sync-backlog` (creates `.devflow/backlog.json` from MODE-A steps)
3. `devflow run --tool local`
4. Loop iterates until `status=complete | blocked | max_iterations_reached`

## Testing Strategy

- Unit test `VCSDriver` with a temp Git repo
- Unit test `Backlog.generate_from_workflow()`
- Unit test `LoopEngine.run()` with mocked agent spawn
- Integration test: full loop over a minimal 2-step workflow

## Constraints

- No changes to `WorkflowEngine.advance()` signature
- No breaking changes to existing CLI commands
- `DEVFLOW_ALLOW_SHELL=1` still required for `command_success` gates
